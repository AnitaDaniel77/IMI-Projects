Lesson 1 — Differentiating UX and UI

The setup: Adrian, co-owner of Little Lemon, noticed the site isn't pulling in orders or reservations. He likes the old look but knows something's broken, so he asks you to redesign it. That redesign runs through the whole course.

The everyday version first

Real life example: Picture a sit-down meal at Little Lemon, not the website.

UX is the whole evening — parking, the waiter getting your order right, food arriving hot, clean bathrooms, whether you'd come back.

UI is only the menu in your hands — its layout, the dish photos, the typeface, how sections are grouped so you find the pasta without hunting.

A beautiful menu can't rescue a two-hour wait for cold food. A flawless kitchen still loses you if the menu's unreadable. You need both.

UX — the experience

UX is the customer's experience with the entire Little Lemon business — staff, service, food, décor, menu, restrooms, plus the site and app. This course narrows it to the website. UX works by asking questions, then answering them inside the product:

  • What do customers actually need?
  • What's stopping them reaching their goal?
  • How intuitive is the site to use?
  • Can they order food easily?
  • Can they move between sections quickly — picking dishes, customising an order?

UI — the interface

UI is what users first see and then interact with. The raw materials:

  • Type and colour
  • Buttons, shapes, icons
  • Images

Good UI groups and arranges these so users reach their goal fast, while it looks beautiful and stays true to the brand.

How they feed each other

UX designer — researches and identifies needs, proposes solutions iteratively through prototypes.

UI designer — creates logical, beautiful layouts and interactive flows informed by a systemised design library.

Careers — company size changes the job

Smaller company: one designer wears every hat — researching needs, designing the solution, and coding it up.

Larger company: roles get specialised.

  • UX Researcher — focuses purely on research.
  • UX Copywriter — owns the content, words, and tone of voice.
  • UI Design-system manager — maintains the repository of interactive elements and components.
  • UI Accessibility specialist — best possible experience for users with impairments.

One-line recap

UX — is the whole journey working? Research, needs, flow, prototypes. The experience.

UI — does it look and feel right to touch? Type, colour, buttons, layout. The surface you interact with.
