Lesson 6 — Empathy Tools: Artifacts in the UX Process

Artifacts are documents design teams create to gather, organise and present information throughout the UX process. The four main ones: user personas, user stories, scenarios and storyboards.

User Personas

A user persona is a fictional character that represents a possible user of your website or app. They keep the design team focused on real end users throughout the entire process.

How to create a persona

  • Conduct customer interviews — interview at least five people. Stop when you're getting no new information.
  • Make assumptions — if you know the product well, use what you know. Back it up with research later.
  • Use web analytics — tells you how users behave but not why. You still need to talk to them.

What a persona includes

  • Header — fictional name, image, demographic profile, a quote summarising what matters most to them
  • Personal background — a mini biography
  • Psychological profile — needs, interests, motivations and pain points

Only include details that influence the final design. No pointless extras.

User Stories

A user story is a brief statement describing who the user is, what they need, and why. One user story per persona.

Template

As a (type of user), I want (a goal) so that (some reason).

Real life example — banking

"As an online banking customer, I want to add a payee to my account so that I can transfer money to the payee."

Little Lemon example

"As a Little Lemon customer, I want to reserve a table online so that I don't have to call during busy hours."

Scenarios

A scenario describes a situation — the user's motivation for being on the site, a question they need answered, and possible ways to achieve their goal. Scenarios are an extension of the user story with added context.

Example

A customer is on their lunch break and wants to reserve a table at Little Lemon for that evening using their phone. They need to know if a table for two is available at 7pm and whether they can request a quiet corner seat.

Scenario mapping process

  • Define the primary user through persona development
  • Identify the critical task they want to achieve
  • Conduct a scenario analysis — contextualise their goals and walk through every step they would take

Storyboards

A storyboard is a visual representation of how a user will interact with a product — sketches, illustrations, screenshots, slideshows or live demos.

How to create one

  • Set the scene — define your persona, the environment, and the plot (what they want to achieve)
  • Sketch out the basic idea for each scene
  • Build it up with as many interactions as needed

Storyboards help teams, stakeholders and users visualise and connect with a design idea before anything is built.

Key takeaway

Document everything you gather. Each artifact — persona, story, scenario, storyboard — backs up your design decisions and keeps the user at the centre of every choice you make.
