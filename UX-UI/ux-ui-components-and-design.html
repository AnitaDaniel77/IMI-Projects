Lesson 17 — Components and Design Systems

What is a component?

A component is any part of your design that can be logically grouped, treated as standalone, and reused. You build small, combine into larger, and assemble those into full interfaces.

Practical example — the Little Lemon header

Instead of rebuilding the header on every page, you create one header component and reuse it. That header itself contains smaller components — a button, a search bar, a navigation bar. Each of those is standalone and can appear elsewhere on the site too.

Think of them as interchangeable blocks. Assemble, reassemble, done. Design and development work gets replicated quickly, and users benefit from consistent patterns that are easier to learn and remember.

What is a design system?

A design system is a set of reusable pre-made components and patterns used to design products at scale. Similar to a brand style guide, it contains rules and best practice behaviours for every design component. Larger companies have very detailed ones — more on that later in the course.

What a design system typically contains

Design guidelines

Vary by business and reflect the brand's underlying values. These govern how components should look and behave.

Design pattern library

A collection of accepted, widely used design patterns. According to the Interaction Design Foundation, a pattern is the recurrence of multiple design elements — shapes, lines, colours — that work together.

UI kit / component library

A collection of UI-focused components: buttons, widgets and more. Teams use these to produce consistent designs quickly without starting from scratch.

Process design guidelines

A collection of accepted patterns that help designers interpret design principles while carrying out a task.

Key takeaway

Components make interfaces consistent and faster to build. Design systems make that consistency scalable across an entire product and team.
