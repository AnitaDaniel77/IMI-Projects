Lesson 39 — Design Systems, UI Kits and Atomic Design

Design system

A design system is everything that goes into creating a product — typography, colours, icons, layouts, grids, coding standards, naming conventions, tone of voice guidelines, style guides and developer documentation. It combines all of these so a team can develop, learn and work together from a single source of truth. Larger design systems include dos and don'ts. Smaller ones may just be a set of components and files.

UI kit

A UI kit is similar to a design system but limited to the elements that populate a user interface — a typographic scale, a colour system, buttons, menu items and components. It shows a systematic approach to design but is less thorough than a full design system. For this course you will use a UI kit.

Design system vs UI kit — quick distinction

A design system covers the whole product including code, content and process. A UI kit covers only the visual and interactive elements users see and touch.

Atomic Design — Brad Frost

Atomic design is a thought process for assembling design elements into larger components, explained through a science metaphor.

Atoms

The smallest individual elements — text boxes, form inputs, links, buttons. Think of these as HTML elements.

Molecules

Atoms combined together. An input box with a label is a molecule. A profile icon with a login link and a sign up button grouped together is a molecule.

Organisms

Groups of molecules working together. A navigation bar is an organism — it contains a logo, links, a feature button, login and sign up molecules, and a search option.

Templates

Low-fidelity wireframes made up of all your atoms, molecules and organisms placed together. The structure without the real content.

Pages

Specific refinements of templates with actual content and images. The final design with real data applied.

Why this matters

Breaking design down this granularly enforces consistency and saves time. A search bar and a login field are almost identical molecules — same structure, different icon and label. Building them as reusable components means you design once and reuse everywhere rather than rebuilding from scratch each time.
