Lesson 15 — Form Design Best Practices

Forms are simple elements that become deeply frustrating when designed without the user in mind. The rules for designing good forms are the same fundamental rules that apply to all UX/UI design — just applied to a smaller scale.

The Little Lemon scenario

After user interviews revealed problems with the ordering form, a prototype was built in HTML and tested with Enzo, an experienced UX/UI designer. His feedback revealed four key problems and their solutions.

Problem 1 — Fields were unclear and too close together

Enzo could not understand what some questions were referring to. Input fields were cramped and hard to distinguish from each other.

Solution:

  • Space input fields consistently — text fields, password fields, checkboxes and radio buttons all need breathing room
  • Use logical grouping — separate payment fields from address fields
  • Apply subtle contrast colour to each row to differentiate sections
  • Ensure labels are clearly associated with their input field

Problem 2 — No visibility of system status or progress

Enzo had no idea whether he was close to completing the form or not. There was no way to go back and review what he had already entered.

Solution:

  • Add a progress indicator so users can see where they are in the flow
  • Allow users to navigate back and review previously entered content
  • Make the path to completion clear at every step

Problem 3 — Validation errors only appeared after full submission

Enzo was only informed of an error after entering all his information. He did not know where the error was or what type of error it was.

Solution:

  • Validate input in real time — as the user fills in each field
  • Display the error message next to the specific field that caused it
  • Use clear, specific language describing exactly what went wrong
  • Language should always match the section of the form the user is in

Problem 4 — No password guidance

The form assumed users knew the password requirements. No instructions were provided, so users had no idea why their password was being rejected.

Solution:

  • Clearly state password requirements upfront — uppercase, lowercase, symbols, numbers
  • Include a password strength indicator that gives immediate feedback as the user types

Key takeaway

Good form design comes down to four things: clear layout with distinguished fields, logical grouping of sections, real-time validation with specific error messages, and upfront guidance so users are never surprised. Align these with both the user's goals and the business goals and you have a form that works.
