Lesson 16 — Form Design Best Practices (Reading)

Expedia found that one unnecessary form field was costing them $12 million in profits per year. That's how much form design matters.

Best practices

Keep it simple

Only include fields you actually need. Every extra field is a reason for a user to leave.

Use one column

Single-column layouts are easier to follow, especially on long multi-step forms. Multiple columns create confusion about which label belongs to which field.

Label the form clearly

Give the form a title that tells users exactly what happens when they submit. "Create an account" is clear. "Submit" alone is not.

Order fields from simplest to hardest

Start with name and email. Once a user commits to filling those in, they're far less likely to abandon the form before reaching billing or shipping details.

Inline validation

Check input in real time as the user types. Show the error immediately next to the field — not after they hit submit. This is what the Little Lemon form got badly wrong.

Left-align all text

Labels above or to the left of fields reduce eye movement. Research from the University of Basel found this directly decreases the time needed to complete a form.

Don't ask for phone numbers unless necessary

Phone number fields create suspicion. Use email instead, or make the phone field optional.

Enable browser autofill

Name fields using words browsers recognise so autofill works. Don't make users type things their browser already knows.

Make it mobile-friendly

Fields, labels and buttons must fit smaller screens without requiring pinch or horizontal scroll.

Write positive error messages

Never blame the user. Be specific about what went wrong and how to fix it. "Please enter a valid email address" beats "Error: invalid input."

Use smart defaults

Pre-fill fields where possible using known context — like the user's current location for city or town. Saves time and reduces errors.

Add progress bars on long forms

Users filling out a multi-step form need to know how far they've come and how far they have to go. A progress bar reduces abandonment.

Use reCAPTCHA not CAPTCHA

CAPTCHA makes users decipher distorted text. reCAPTCHA just asks them to check a box or identify elements in an image — far less friction for the same security benefit.

Support tab navigation

Users should be able to move between fields using the tab key without touching their mouse. Basic keyboard accessibility.
