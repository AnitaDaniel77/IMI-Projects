Lesson 28 — Grid Systems

Every webpage is built from squares and rectangles sitting inside a grid. The grid is invisible to users but does the work of organising content, creating alignment and giving the interface a consistent rhythm.

Parts of a grid

Columns run vertically and are where content sits. Standard counts are 12 on desktop, 8 on tablet, 4 on mobile. Most column widths fall between 60-80px. Gutters are the gaps between columns, typically 20px. Margins are the space between the outermost columns and the frame edge. Modules are the units created where rows and columns intersect — each design element (text, image, button) fits into one or more modules.

Types of grids

Block grids are the simplest: a single full-width column. Good for hero images or long-form text that doesn't need to sit beside anything else.

Column grids are the most common. Multiple vertical columns organise content side by side. The column count changes per device but the column width generally stays fixed.

Modular grids combine columns and rows into equal cells. App home screens and e-commerce category pages are the clearest examples — a grid of products or icons that all occupy the same amount of space.

Hierarchical grids are freestyle. Elements span across columns irregularly based on content priority rather than a fixed structure. News homepages use this — a lead story gets six columns, secondary stories get two each.

Baseline grids set consistent vertical rhythm for text by aligning line heights across the page. Think of ruled paper. The 8-point grid from earlier lessons is a baseline grid applied to spacing.

Using grids well

Set your grid before you start designing, not after. Trying to retrofit a grid onto a finished layout is painful. Once it's set, you won't have to think about spacing decisions again — the grid makes them for you.

Don't fill every module. Whitespace isn't wasted space, it's what lets users breathe between sections. Large margins and healthy gaps between content blocks make a page feel considered rather than cluttered.

The 960 grid system is a practical starting point for desktop: 960px total width, 12 columns at 60px each, 10px spacing on each side of every column for a 20px total gutter, 940px content area. It's been around long enough that most designers have some instinct for it already.

Breaking the grid is fine once you understand it. The grid is a guide, not a cage. The designers who break it intentionally and effectively are the ones who understood it first.
