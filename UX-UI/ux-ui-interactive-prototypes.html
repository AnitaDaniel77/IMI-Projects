Lesson 42 — Interactive Prototypes in Figma

A prototype is an almost-working model of the product used to test with users and stakeholders before the final product is built. Interactive prototypes show how a design will behave and help others understand what the finished product will look and feel like.

Tilly's user flow — the scenario being prototyped

Tilly is hungry after work. She opens the Little Lemon mobile site to order online. She sees the bruschetta, taps it for more details, decides she likes it as is and adds it to her order. An order summary appears. She spots the Greek salad, taps it, reads the details, adds it too. The summary updates and the basket shows two items. She taps back to browse more menu categories.

This flow covers five screens:

  • Screen 1 — home / menu listing
  • Screen 2 — bruschetta detail page
  • Screen 3 — order summary with bruschetta
  • Screen 4 — Greek salad detail page
  • Screen 5 — updated order summary with both items

How to link screens in Figma

Switch to Prototype mode in the right panel. A blue box with an arrow appears on the selected frame. Blue circular icons appear on elements inside the frame that can be linked.

To create a link: double-click into the frame until the specific element is highlighted (for example the bruschetta image). Click the blue circle that appears on its right edge and drag it to the destination frame. The interaction details appear in the right panel — by default it says Navigate to and shows the destination screen name.

The links for this prototype

  • Bruschetta element on Screen 1 → Screen 2 (bruschetta detail)
  • Add button on Screen 2 → Screen 3 (order summary)
  • Greek salad element on Screen 3 → Screen 4 (Greek salad detail)
  • Add button on Screen 4 → Screen 5 (updated order summary)
  • Back button on Screen 5 → Screen 1 (home)

Testing the prototype

Click the blue play button on the Figma canvas to open the prototype view. Tap through the flow as Tilly would — bruschetta detail, add to basket, order summary, Greek salad detail, updated summary, back to home. If all links work the prototype is ready to share with Adrian and users for feedback.

Key takeaway

Prototypes are about linking screens to simulate real interactions. The design doesn't need to be perfect — it needs to be testable. Build the flow, test it with real users, collect feedback and iterate.
