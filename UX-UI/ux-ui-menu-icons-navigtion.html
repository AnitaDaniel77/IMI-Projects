Lesson 20 — Menu Icons in Navigation

Icons are small graphics representing a program, file or function. Navigation icons specifically help users find content in an app or website. There's no official naming system across the industry — Font Awesome and Material Design use different names for the same icons — but these are the ones you'll encounter most often.

The icons

Hamburger menu

Three horizontal lines of equal length stacked on top of each other. Named because it looks like two buns and a patty. Signals there's more content beyond what's visible. Works well on mobile — avoid it on desktop where space allows full navigation to show.

Kebab menu

Three vertical dots, originally from Google's Material Design. Opens a menu with additional options. Usually sits in the upper right or left corner of a screen or window.

Meatballs menu

Three horizontal dots. Same purpose as the kebab but horizontal, making it a better fit for tables and other horizontally oriented elements. Typically found in the upper-right corner.

Bento menu

A grid of dots or squares, named after bento boxes. Used to switch between different applications, solutions or functions within the same product — think the Google apps grid in Gmail's top right corner.

Döner menu

Three horizontal lines of varying lengths stacked vertically, resembling the popular Turkish dish. The go-to icon for filters and sort options in modern apps. Often paired with "sort by" controls.

Others

Candy box, cheeseburger, veggie burger and more exist. If you use one of these less common icons, make sure the context makes its meaning obvious — don't assume users will recognise it.

What this means for Little Lemon

The current site gave users no clear navigation cues — "Menu" looked like body text, not a tappable element. A hamburger icon in the top corner would immediately signal to mobile users that navigation is available, removing one of Tilly's biggest frustrations.
