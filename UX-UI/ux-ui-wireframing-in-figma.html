Lesson 30 — Wireframing in Figma

A wireframe is a basic structure for each screen before branding, colours and images are added. It communicates ideas quickly and focuses entirely on what the user needs to complete a task. No colour, no fonts, no real images — just layout and functionality.

Mobile first design

Start with the mobile version before scaling up to desktop. Most users access websites on mobile, and it's easier to solve design problems on a small screen and adapt upward than to squeeze a desktop layout down into a phone.

Little Lemon — content requirements

Before drawing anything, list what needs to appear on the screens. These become your content blocks:

  • Navigation bar
  • Little Lemon brand and about section
  • Menu categories and dish descriptions
  • Prices and photos of dishes
  • Customise order section — number of dishes per item
  • Delivery or take-out options
  • Add to order button
  • Login and pay section

Building the wireframe in Figma

Select a mobile frame and apply a 4-column grid. Draw rectangles for each content block — these are your placeholders. Move them outside the frame first to use as a reference, then build each section inside the frame using them as a guide.

Navigation bar

Logo on the left — draw a rectangle with diagonal lines through it to indicate an image placeholder. This is the conventional wireframe way to show images. Add a hamburger menu icon and a basket icon. Tapping the basket opens an overlay showing current order items and a checkout option. Tapping the hamburger opens a navigation overlay to other pages.

About section

Restaurant name, city, a brief description, an image placeholder, and a "Book a table" button that links to another page.

Menu categories

Represented as buttons labelled Cat 1, Cat 2, Cat 3 — categories haven't been finalised yet so placeholders keep the wireframe moving.

Dish listings

Each dish needs: title, dish name, description, price and image placeholder. This section scrolls vertically. Tapping a dish takes the user to a detail screen. Adrian wants this week's specials highlighted first.

Key wireframing rules

  • No colour, no real images, no final fonts
  • Use rectangles with diagonal lines for image placeholders
  • Label placeholder buttons clearly (Cat 1, not "button")
  • Note interactions — what happens when something is tapped
  • Focus on layout and functionality only
