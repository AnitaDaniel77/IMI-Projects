DJ Lopes Website — Header Component

What this component is

The Header is the sticky bar at the top of every screen: logo on the left, navigation links in the middle, social icons on the right, plus a hamburger menu on small screens. It lives in its own file, src/components/Header.jsx, and is imported into App.jsx. Keeping it separate means we can edit it without touching the rest of the site.

Icons: react-icons instead of lucide-react

We used react-icons because it includes brand logos for Instagram, Facebook and TikTok. Lucide, a common alternative, removed its brand icons, so it could not give us a TikTok logo. Install once with npm install react-icons, then import only the icons you need.

```jsx import { FaInstagram, FaFacebookF, FaTiktok, FaBars, FaXmark } from 'react-icons/fa6' ```

Storing links as data, then mapping over them

The nav links and social links are kept as arrays of objects, then .map() turns each one into a tag. Write the list once and render from data, so adding or removing a link means editing the array, not copy-pasting JSX.

```jsx const navLinks = [ { label: 'Home', href: '#home' }, { label: 'Services', href: '#services' }, ] ``` ```jsx {navLinks.map((link) => ( {link.label} ))} ```

The key prop gives React a stable id for each list item so it updates efficiently. React warns if you forget it.

Why the icon key is capitalised

Each social object stores its icon under a capital-I key, Icon. React only treats a JSX tag as a component when its name starts with a capital letter. A lowercase tag name is read as a plain HTML element and would not render the React icon. That is why the rendered tag is capitalised.

```jsx {socials.map(({ label, href, Icon }) => ( ))} ```

The mobile menu: useState and conditional rendering

useState(false) holds whether the menu is open. The hamburger button flips it with setMenuOpen. The dropdown is wrapped in a menuOpen check, meaning it only renders when menuOpen is true, the same conditional rendering pattern from Advanced React.

```jsx const [menuOpen, setMenuOpen] = useState(false) ``` ```jsx {menuOpen && (
...mobile links...
)} ```

Responsive design with Tailwind prefixes

Tailwind prefixes like md: apply a class only from a screen width up. hidden md:flex means hidden on phones, visible from tablet width. md:hidden is the reverse, used for the hamburger so it only shows on small screens. One component serves both phone and desktop layouts.

Sticky header and smooth scroll

sticky top-0 z-50 keeps the header pinned to the top as you scroll, above other content. The nav links point to section ids like #services and #about. Adding scroll-behavior smooth to the html element in index.css makes clicking a link glide to that section instead of jumping.

```css html { scroll-behavior: smooth; } ```

Lesson: pasting code into Git Bash

Pasting multi-line code into Git Bash (MinTTY) can silently drop blank lines and very short lines, such as a line containing only an opening anchor tag. This broke the build twice. The fix was to write each anchor tag on one line with its attributes and content together, so no line was short enough to be dropped. If a paste fails again, run cat on the file and look for missing tags.

Note: logo shows a white box

The wordmark is a JPG, and JPGs cannot have transparent backgrounds, so it sits in a small white rectangle on the dark header. A transparent PNG export of the wordmark removes the box. Cosmetic only.
