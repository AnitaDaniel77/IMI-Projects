DJ Lopes Website — Project Setup

Why React + Vite + Tailwind?

Real life example: Like building a DJ booth. Vite is the fast-assembling rig (quick dev server, instant reload). React is the modular equipment — reusable components (Header, Hero, BookingCalendar) you can plug in anywhere. Tailwind is the lighting design system — pre-built utility classes instead of writing custom CSS from scratch every time.

Step 1 — Create the React app with Vite

```bash npm create vite@latest dj-lopes-app -- --template react cd dj-lopes-app npm install ```

This scaffolds a fast, modern React project. Vite replaced the older create-react-app tool because it's much faster during development.

Step 2 — Install Tailwind CSS (v4 — newer setup)

```bash npm install -D tailwindcss postcss autoprefixer npm install @tailwindcss/vite ```

Note: Tailwind v4 removed the old `npx tailwindcss init` command and the `tailwind.config.js` file that most tutorials show (those are for v3). Instead, v4 uses a Vite plugin directly.

Step 3 — Register the Tailwind plugin in vite.config.js

```jsx import { defineConfig } from 'vite' import react from '@vitejs/plugin-react' import tailwindcss from '@tailwindcss/vite' export default defineConfig({ plugins: [react(), tailwindcss()], }) ```

This tells Vite to process Tailwind's utility classes while building the app.

Step 4 — Import Tailwind in index.css

```css @import "tailwindcss"; ```

This single line replaces ALL of the old `@tailwind base; @tailwind components; @tailwind utilities;` lines used in Tailwind v3. It pulls in every utility class Tailwind offers.

Understanding Tailwind class names

Instead of writing custom CSS rules, Tailwind gives pre-made utility classes you combine directly on the element. Faster to write, no separate CSS file needed for most things.
