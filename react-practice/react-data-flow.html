## Parent-Child Data Flow in React

Real life example:
Like a head office sending a discount update to all branches.
Change it once at head office (parent) and all branches
(children) update automatically.

--------------------------------------------
PARENT-CHILD RELATIONSHIP
--------------------------------------------

Parent component  = Promo
Child component   = PromoHeading

/* Parent — single source of truth */
const data = {
    heading: "99% of all items",
    callToAction: "Everything must go"
}

function Promo() {
    return (
        <div>
            {/* Pass data down to child using props */}
            <PromoHeading
                heading={data.heading}
                callToAction={data.callToAction}
            />
        </div>
    )
}

/* Child — receives and displays props */
function PromoHeading(props) {
    return (
        <div>
            <h1>{props.heading}</h1>
            <h2>{props.callToAction}</h2>
        </div>
    )
}

--------------------------------------------
KEY PRINCIPLES
--------------------------------------------

1. Single source of truth
   Store data in parent — change once, updates everywhere

2. DRY — Don't Repeat Yourself
   Never duplicate the same data in multiple components

3. Unidirectional data flow
   Data flows parent → child only, never child → parent

4. Props pass the data
   Parent sends data via props
   Child receives and displays it

--------------------------------------------
WHY THIS MATTERS
--------------------------------------------

Without single source of truth:
- Update text in 3 components separately
- Risk of typos
- Boss changes discount again = 3 more updates

With single source of truth:
- Update data object once in parent
- All child components update automatically
- No risk of inconsistency

DJ LOPES WEBSITE EXAMPLE
--------------------------------------------

/* Parent — stores event data */
const eventData = {
    name: "Afrobeats Night",
    price: "R150",
    venue: "Club Havana"
}

function EventPage() {
    return (
        <div>
            <EventCard
                name={eventData.name}
                price={eventData.price}
                venue={eventData.venue}
            />
            <EventBanner
                name={eventData.name}
            />
            <EventFooter
                name={eventData.name}
                venue={eventData.venue}
            />
        </div>
    )
}
/* Change eventData once — all three components update */

## Data Flow in React — Reading Notes

Data flows ONE WAY only:
Root → Parent → Child → Grandchild

Props are IMMUTABLE — cannot be changed by the child

Benefits:
1. Easier to understand component logic
2. Simpler to debug — always know where data came from

--------------------------------------------
MULTI-LEVEL DATA FLOW EXAMPLE
--------------------------------------------

/* Parent — Dog */
function Dog() {
    return (
        <Puppy
            name="Rex"
            bowlShape="round"
            bowlStatus="full"
        />
    )
}

/* Child — Puppy receives all props, passes some down */
function Puppy({ name, bowlShape, bowlStatus }) {
    return (
        <div>
            <p>{name}</p>
            {/* Passes only bowl props to grandchild */}
            <Bowl
                bowlShape={bowlShape}
                bowlStatus={bowlStatus}
            />
        </div>
    )
}

/* Grandchild — Bowl receives only what it needs */
function Bowl({ bowlShape, bowlStatus }) {
    return (
        <div>
            <p>Shape: {bowlShape}</p>
            <p>Status: {bowlStatus}</p>
        </div>
    )
}

--------------------------------------------
KEY RULES
--------------------------------------------

Props flow down   → parent to child to grandchild
Props are fixed   → child cannot modify received props
Change at source  → update parent, children update automatically
One direction     → never flows back up
