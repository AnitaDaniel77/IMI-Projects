Controlled Components and Forms

Real life example: Like a hotel receptionist who controls the guest book. Instead of guests writing anything they want (DOM controls), the receptionist validates and controls every entry (React controls).

What is a controlled component?

A component where React state is the single source of truth for the form input value — not the DOM.

Basic controlled input

```jsx import { useState } from 'react' function ReservationForm() { const [name, setName] = useState('') function handleChange(event) { // event.target.value = what user typed setName(event.target.value) } function handleSubmit(event) { // prevents default HTML form submission event.preventDefault() console.log("Name submitted:", name) } return (
) } ```

How it works step by step

```jsx // 1. User types a character // 2. onChange fires — handleChange runs // 3. setName updates state with new value // 4. React re-renders — input shows updated value // 5. React is always in control of what is displayed ```

Controlled vs Uncontrolled

```jsx // Controlled — React controls the value // Uncontrolled — DOM controls the value ```

DJ Lopes website example

```jsx import { useState } from 'react' function BookingForm() { const [form, setForm] = useState({ name: '', event: '', date: '' }) function handleChange(event) { const { name, value } = event.target setForm({ ...form, [name]: value }) } function handleSubmit(event) { event.preventDefault() console.log("Booking submitted:", form) } return (
) } ```

Key points

value prop — React controls what is displayed

onChange — updates state on every keystroke

onSubmit — handles form submission

event.preventDefault() — stops default HTML form behaviour
