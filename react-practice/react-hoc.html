Higher-Order Components (HOC)

What are cross-cutting concerns?

Real life example: Like building security at an office complex. Every building (component) needs the same security checks, but security isn't part of any single business's work. It's shared infrastructure.

Cross-cutting concerns: permissions, error handling, logging, data subscriptions — needed everywhere but not part of business logic.

What is a HOC?

A function that takes a component and returns a new component with extra capabilities.

```jsx // Component transforms props → UI // HOC transforms a component → another component function withSubscription(WrappedComponent, selectData) { return function(props) { const [data, setData] = useState(selectData(dataSource)) useEffect(() => { function handleChange() { setData(selectData(dataSource)) } dataSource.addChangeListener(handleChange) return () => dataSource.removeChangeListener(handleChange) }, []) // pass data + other props through to wrapped component return } } ```

Key points

HOC = function that takes a component, returns a new component

Encapsulates shared logic in one place

Wrapped components stay stateless and unchanged

Convention — pass through other props to wrapped component

HOC Best Practices and Caveats

1. Never mutate the wrapped component

```jsx // WRONG — mutating the original component const HOC = (WrappedComponent) => { WrappedComponent = () => { } // never do this } // CORRECT — return a NEW component, never mutate const HOC = (WrappedComponent) => { return (props) => { return } } ```

2. Always pass unrelated props through

```jsx // CORRECT — spreads all original props plus injects new one const withMousePosition = (WrappedComponent) => { const injectedProp = { mousePosition: { x: 10, y: 10 } } return (originalProps) => { return ( ) } } ```

3. Maximize composability — Component => Component

```jsx // Currying pattern — common in Redux const EnhancedComponent = connect(selector, actions)(WrappedComponent) // Broken down const HOC = connect(selector, actions) const EnhancedComponent = HOC(WrappedComponent) // Compose multiple HOCs together const enhance = compose( withMousePosition, withURLLocation, connect(selector) ) const EnhancedComponent = enhance(WrappedComponent) ```

Caveat 1 — never create HOC inside another component

```jsx // WRONG — creates new component every render, loses state const Component = (props) => { const EnhancedComponent = HOC(WrappedComponent) // never do this return } // CORRECT — create HOC outside component scope, once const EnhancedComponent = HOC(WrappedComponent) const Component = (props) => { return } ```

Caveat 2 — refs are not passed through

```jsx // ref points to outermost HOC, not the wrapped component // Solution: use React.forwardRef ```

Key takeaways

Never mutate the wrapped component — always return a new one

Always pass through unrelated props with spread

Create enhanced components outside other components

Use React.forwardRef when refs need to reach the wrapped component
