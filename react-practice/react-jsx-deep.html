JSX, Elements and the Declarative Model

What is JSX?

Real life example: Like an architect's blueprint. It looks like the final building (HTML) but it is actually a set of instructions (JavaScript) that tells React what to build.

Components vs Elements

```jsx // Component — a function that returns JSX function SubmitButton() { return } // Element — a plain JavaScript object describing what to render { type: 'button', // DOM tag or component name props: { children: 'Submit' } } ```

Virtual DOM

Key points

Elements are plain JavaScript objects — cheap to create and compare

Components and DOM nodes can be mixed in the element tree

React keeps repeating until all components resolve to DOM tags

This is what makes React fast — diffing objects is much faster than diffing the real DOM

Types of Children in JSX

JSX elements as children

```jsx // Nested components <Body /> </Alert> // Mix string and JSX elements <Alert> <div>Are you sure?</div> <Body /> </Alert> ``` <h2>React Fragments — group without extra DOM node</h2> ```jsx // Long syntax return ( <React.Fragment> <li>Pizza margarita</li> <li>Pizza diavola</li> </React.Fragment> ) // Short syntax — preferred return ( <> <li>Pizza margarita</li> <li>Pizza diavola</li> </> ) ``` <h2>JavaScript expressions as children</h2> ```jsx // Expression in curly brackets <MyComponent>{'Little Lemon'}</MyComponent> // map() to render list function List() { const desserts = ['tiramisu', 'ice cream', 'cake'] return ( <ul> {desserts.map(dessert => ( <Item key={dessert} title={dessert} /> ))} </ul> ) } // Mix expression with text function Hello({ name }) { return <div>Hello {name}!</div> } ``` <h2>Booleans, null and undefined — render nothing</h2> ```jsx // All render the same empty div <div>{false}</div> <div>{null}</div> <div>{undefined}</div> <div>{true}</div> // Useful for conditional rendering <div>{showModal && <Modal />}</div> ``` <h2>Common gotcha — 0 still renders</h2> ```jsx // WRONG — renders 0 when array is empty {props.desserts.length && <DessertList />} // CORRECT — ensure expression is always boolean {props.desserts.length > 0 && <DessertList />} {!!props.desserts.length && <DessertList />} ```
