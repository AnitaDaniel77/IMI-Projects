React State

What is State?

State is a component's internal data that determines its current behaviour.

Real life example: Like an alarm clock — on, off, snooze are built-in modes. Push a button to change the state.

Props vs State

Props = data from outside, parent controls it, read only

State = data from inside, component controls it, can change

Props vs State — Key Differences

Real life example: Like restaurant blueprints. The blueprint (component) is the same for all locations. Props are the specific configurations each manager chooses (tables, chairs). State is what changes during the day (number of customers seated right now).

Working with Objects in useState

Real life example: Like updating a contact card. Don't rewrite the whole card just to change the phone number — copy it first, then update only what changed.

Basic object state

```jsx const [greeting, setGreeting] = useState({ greet: "Hello, World" }) ```

WRONG — cannot reassign const

```jsx // TypeError: Assignment to constant variable greeting = { greet: "Hello, World-Wide Web" } setGreeting(greeting) ```

WRONG — mutating state directly

```jsx // No error but UI never updates greeting.greet = "Hello, World-Wide Web" setGreeting(greeting) ```

CORRECT — spread operator to copy first

```jsx function updateGreeting() { const newGreeting = { ...greeting } // copy the object newGreeting.greet = "Hello, World-Wide Web" // update copy setGreeting(newGreeting) // set the new copy } ```

CORRECT — update one property using arrow function

```jsx const [greeting, setGreeting] = useState({ greet: "Hello", place: "World" }) function updatePlace() { // prevState = previous value of greeting setGreeting(prevState => ({ ...prevState, // keep all existing properties place: "World-Wide Web" // only update place })) } ```

Key rule

Never mutate state directly. Always copy with spread operator first, then update the copy.
