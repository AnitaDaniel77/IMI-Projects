Testing React Components

Real life example: Like a factory testing every product before it leaves the line. Catching bugs before delivery saves time, money, and customer complaints.

Why testing matters

Catches bugs before they reach users. Saves time and money. Builds confidence that the app works as intended.

Best practices

  • Test behavior, not implementation details. Focus on what the user sees and does, not how the component is built.
  • Use descriptive test names. Clearly state what the test is verifying.
  • Keep tests isolated. Each test should only verify one thing.
  • Use test IDs or accessible roles to select elements. Avoid relying on class names or DOM structure.

    • Two tools — Jest and React Testing Library

    Basic test example

    ```jsx import { render, screen } from '@testing-library/react' import App from './App' test('renders Little Lemon Restaurant link', () => { // Step 1 — render the component render() // Step 2 — query the DOM for text const linkElement = screen.getByText(/Little Lemon Restaurant/i) // Step 3 — assert it exists in the document expect(linkElement).toBeInTheDocument() }) ```

    How it works

    1. Render the component into a virtual DOM. 2. Query the virtual DOM for elements. 3. Assert that the elements exist or have certain properties.

    Key points

    Mocking — replacing complex functions with simpler simulated versions for isolated testing

    Tests should read like how a real user interacts with the app

    A failing test that catches a real bug is exactly what testing is for

    Testing a Feedback Form — Practical Example

    Real life example: Like a security guard checking ID before letting someone through a door. The test checks the form refuses entry (submission) when requirements aren't met.

    Scenario

    Submit button disabled if score is below 5 AND comment is less than 10 characters. Test verifies this logic works correctly.

    Test file convention

    Test files are named with a .test.js or .spec.js extension and placed in a __tests__ directory alongside the component they test.

    Full test example

    ```jsx import { render, screen, fireEvent } from '@testing-library/react' import FeedbackForm from './FeedbackForm' test('prevents submission when score is low and comment is missing', () => { // Step 1 — create mock function to track calls const mockSubmit = jest.fn() // Step 2 — render component with mock as prop render() // Step 3 — find the range input by its label const scoreInput = screen.getByLabelText(/score/i) // Step 4 — simulate changing the input value fireEvent.change(scoreInput, { target: { value: 3 } }) // Step 5 — find the submit button by its role const submitButton = screen.getByRole('button') // Step 6 — simulate clicking the button fireEvent.click(submitButton) // Step 7 — assert mockSubmit was NOT called expect(mockSubmit).not.toHaveBeenCalled() // Step 8 — assert button is disabled expect(submitButton).toHaveAttribute('disabled') }) ```

    Key React Testing Library utilities

    render — renders a component into a virtual DOM for testing

    screen — provides methods to query the virtual DOM for elements

    fireEvent — simulates user interactions like clicks and input changes

    jest.fn() — creates a mock function to track calls and arguments

    expect — assertion library to verify conditions in tests
