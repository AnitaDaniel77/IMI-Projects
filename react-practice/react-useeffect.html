What is a side effect?

Real life example: Like a chef who cooks a meal (pure function) but also calls the supplier to reorder ingredients (side effect). The cooking always produces the same dish, but the phone call reaches outside the kitchen.

Pure vs Impure functions

```jsx // PURE — same input always gives same output function EstablishedYear({ year }) { return

Established year: {year}

} // IMPURE — console.log is a side effect function ShoppingCart({ total }) { console.log(total) // reaches outside the function return

Total: {total}

} ```

Side effects include

useEffect — contains side effects properly

```jsx import { useEffect } from 'react' function ShoppingCart({ total }) { // side effect contained inside useEffect useEffect(() => { console.log(total) }, []) return

Total: {total}

} ```

useEffect syntax

```jsx useEffect( () => { }, // callback — contains the side effect [] // dependency array — controls when effect runs ) // runs once on mount useEffect(() => { }, []) // runs when count changes useEffect(() => { }, [count]) // runs after every render useEffect(() => { }) ```

Key points

Pure function — same input, same output, no side effects

Impure function — has side effects, reaches outside itself

useEffect — the correct place to put side effects in React

Dependency array — controls when the effect runs

useEffect — Dependency Array and Cleanup

Three ways to use the dependency array

```jsx // Runs after EVERY render — no array useEffect(() => { document.title = 'Little Lemon' }) // Runs ONCE on mount — empty array useEffect(() => { document.title = 'Little Lemon' }, []) // Runs when version changes — specific dependency useEffect(() => { document.title = `Little Lemon v${version}` }, [version]) ```

Multiple effects — separate concerns

```jsx function MenuPage({ id }) { // Effect 1 — sets page title useEffect(() => { document.title = 'Little Lemon' }, []) // Effect 2 — fetches menu data useEffect(() => { fetch(`https://littlelemon/menu/${id}`) .then(res => res.json()) .then(data => setData(data)) }, [id]) } ```

Cleanup — return a function to free memory

Real life example: Like subscribing to a magazine. When you move house (component unmounts) you cancel the subscription (cleanup) so magazines stop arriving.

```jsx function LittleLemonChat({ chatId }) { const [status, setStatus] = useState('offline') useEffect(() => { // subscribe when component mounts LemonChat.subscribeToMessages(chatId, () => setStatus('online')) // cleanup — runs when component unmounts return () => { setStatus('offline') LemonChat.unsubscribeFromMessages(chatId) } }, [chatId]) } ```

Key points

No array — runs after every render

Empty array — runs once on mount

With dependencies — runs when those values change

Return function — cleanup when component unmounts

Use multiple useEffect calls to separate unrelated logic

Fetch and Asynchronous JavaScript

Real life example: Like a post office clerk who doesn't wait for one task to finish before starting the next. They delegate to other clerks (browser APIs) and keep serving the queue (executing code).

Why asynchronous matters

```jsx // JavaScript is single-threaded — can only do one thing at a time // Fetch delegates the server request to the browser API // JavaScript keeps running while waiting for the response console.log('another customer approaching') // runs first fetch('https://randomuser.me/api/') // delegates to browser API .then(res => res.json()) .then(data => console.log(data)) // runs last — when data arrives console.log('our valued customer, please wait') // runs second ```
